# July08Application 학습 정리

**날짜:** 2024년 7월 8일

## 1. 프로젝트 개요
안드로이드의 기본적인 UI 구성 요소와 이벤트 처리, 그리고 다양한 대화상자(Dialog) 및 선택기(Picker) 활용 방법을 학습하기 위한 프로젝트입니다. 프로젝트는 `app`, `picker`, `pickerdialog` 세 개의 모듈로 구성되어 있습니다.

## 2. 주요 학습 내용

### 1) View Binding 활용
- 기존의 `findViewById` 대신 `View Binding`을 사용하여 레이아웃 내의 뷰에 안전하고 직관적으로 접근하는 방법을 익혔습니다.
- 각 모듈의 `build.gradle.kts`에서 `viewBinding = true` 설정을 통해 활성화하였습니다.

### 2) 액티비티 전환 및 계산기 구현 (`app` 모듈)
- `Intent`를 사용하여 `MainActivity`에서 `CalcActivity`로 전환하는 방법을 학습했습니다.
- 간단한 사칙연산 기능을 구현하며 안드로이드 로직 처리를 연습했습니다.

### 3) 날짜 및 시간 선택기 (Pickers)
- **`picker` 모듈**: XML 레이아웃 파일에 `DatePicker`와 `TimePicker` 위젯을 직접 포함시켜 화면의 일부로 구성하는 방법을 학습했습니다.
- **`pickerdialog` 모듈**: 버튼 클릭 시 `DatePickerDialog`와 `TimePickerDialog`를 팝업 형태로 띄워 사용자로부터 날짜와 시간을 입력받는 방식을 구현했습니다. `Calendar` 클래스를 이용해 현재 시간을 초기값으로 설정하는 방법을 익혔습니다.

### 4) 대화상자 (AlertDialog)
- `AlertDialog.Builder`를 사용하여 사용자에게 알림을 전달하거나 '확인/취소' 버튼을 통해 의사를 확인하는 대화상자 구현 방법을 학습했습니다.
- `.run { ... }` 블록 또는 체이닝 방식을 이용한 빌더 패턴 사용법을 익혔습니다.

### 5) 최신 디자인 가이드 적용
- `enableEdgeToEdge()`를 호출하여 앱의 UI가 시스템 바 영역까지 확장되도록 설정하는 최신 안드로이드 디자인 적용 방식을 확인했습니다.

## 3. 요약
오늘은 안드로이드 앱 개발의 기초가 되는 **이벤트 리스너 등록, 뷰 바인딩, 액티비티 이동, 그리고 다양한 다이얼로그 활용**에 대해 집중적으로 학습하였습니다. 특히 상황에 따라 위젯 형태의 Picker와 Dialog 형태의 Picker 중 적절한 것을 선택하여 구현하는 능력을 길렀습니다.
