# [프로젝트 제안서] 모바일 프로그래밍 프로젝트

## 📌 프로젝트 개요
* **과제명:** 안드로이드 스튜디오를 활용한 '만개의 레시피' 클론 코딩 및 기능 고도화
* **소속:** 정보통신공학과
* **제출자:** 전지훈

---

## 1. 프로젝트 선정 배경 및 이유

### 🔹 실제 자취생의 현실적인 고민 해결
곧 독립하여 자취를 시작하게 되면서 매일 마주하게 될 가장 큰 현실적인 문제는 "오늘 무엇을 먹을 것인가"와 "요리를 어떻게 할 것인가"라고 생각했습니다. 요리 초보자로서 배달 음식에만 의존하기보다 직접 건강한 한 끼를 해결하고 싶다는 강력한 니즈가 생겼으며, 이 과정에서 친숙하게 도움을 받을 수 있는 서비스가 필요했습니다.

### 🔹 실제 유저 경험(UX) 기반의 접근
주방에서 실제로 요리를 할 때 여러 번 '만개의 레시피' 앱을 직접 사용해 보았습니다. 요리 초보자의 눈높이에 맞춘 직관적인 UI와 단계별 설명이 얼마나 큰 도움이 되는지 몸소 경험했습니다. 제가 직접 겪은 편리함과 아쉬운 점을 바탕으로 클론 코딩을 진행한다면, 유저 인터페이스(UI)와 사용자 경험(UX)을 설계할 때 훨씬 더 깊이 있게 고민하고 개선점을 찾아낼 수 있겠다고 판단했습니다.

### 🔹 개발자 관점의 기술적 훈련
단순히 요리 정보를 보여주는 것에 그치지 않고, 초보 자취생에게 꼭 필요한 **'냉장고 털기(재료 매칭 알고리즘)'**나 **'장보기 메모(로컬 DB CRUD)'** 같은 핵심 기능을 모바일 환경에 맞게 직접 구현해 봄으로써, 데이터 처리 및 아키텍처 설계 능력을 기르기에 최적의 주제라고 생각하여 선정했습니다.

## 2. '만개의 레시피' 앱 현황 및 검증 지표

본 프로젝트의 대상인 '만개의 레시피'는 다음과 같은 시장 지표를 지닌 검증된 플랫폼으로, 모바일 프로그래밍 레이아웃의 표준을 학습하기에 가장 이상적인 모델입니다.

* **대한민국 No.1 요리 플랫폼:** 국내에서 서비스 중인 요리/조리 카테고리 앱 중 독보적인 인지도와 점유율 1위를 차지하고 있는 국민 요리 앱입니다.
* **구글 플레이스토어 1,000만+ 다운로드:** 누적 다운로드 수 1,000만 건을 돌파하여, 모바일 환경에서의 인터랙션 안정성과 대중성이 이미 완벽하게 검증된 서비스입니다.
* **20만 개 이상의 방대한 레시피 데이터:** 대규모 데이터 세트를 그리드(Grid) 레이아웃과 대용량 리스트 뷰(`RecyclerView`)로 효율적으로 다루는 대용량 데이터 중심의 UI 디자인을 벤치마킹할 수 있습니다.
* **정형화된 UI/UX의 교과서:** 상단 롤링 배너, 퀵 카테고리 메뉴, 하단 탭 네비게이션 구조는 안드로이드 애플리케이션의 '가장 정석적이고 가독성 높은 레이아웃'을 학습하기에 최고의 텍스트북입니다.

## 3. 최종 화면 구성 및 컴포넌트 설계 (총 11개 화면)

구현 단계에서 완성도를 높이기 위해 하단의 5대 메인 탭(Fragment)과 기능 연동형 하위 화면(Activity)을 유기적으로 연결하여 총 11개의 화면으로 설계합니다.

| 번호 | 화면 명칭 | 컴포넌트 타입 | 핵심 구현 내용 및 기술 요소 |
| :--- | :--- | :--- | :--- |
| **1** | **홈 화면** | Fragment | `ViewPager2` 기반 롤링 배너, 퀵 메뉴 아이콘, 가로 스크롤 `RecyclerView` |
| **2** | **분류 화면** | Fragment | `GridLayoutManager`를 활용한 바둑판 배열 형태의 격자형 카테고리 메뉴 |
| **3** | **혜택 화면** | Fragment | 마이 포인트 현황 및 이벤트 배너 대시보드 구조 설계 |
| **4** | **스크랩 화면** | Fragment | `TabLayout` + `ViewPager2` 연동을 통한 즐겨찾기 레시피 폴더별 리스트 |
| **5** | **MY 화면** | Fragment | 프로필 영역 구성 및 개인 맞춤형 서브 메뉴 리스트 스크롤 뷰 구현 |
| **6** | **통합 검색 화면** | Activity | 홈 화면 상단 검색창 연동, 검색어 기반 리스트 실시간 필터링 및 결과 표시 |
| **7** | **레시피 상세 화면** | Activity | 요리 개요, 필수 재료 리스트, 세로 스크롤 기반 단계별 요리 순서 뷰 |
| **8** | **냉장고 털기 화면** | Activity | **[핵심 알고리즘]** `ChipGroup` 기반 재료 선택 및 교집합 매칭 정렬 로직 |
| **9** | **장보기 메모 화면** | Activity | **[로컬 DB 연동]** 안드로이드 `Room` DB를 활용한 재료 체크리스트 CRUD |
| **10**| **내가 본 레시피 화면**| Activity | 사용자가 최근 클릭하여 방문한 레시피 데이터의 히스토리 스택 리스트 표시 |
| **11**| **레시피 등록 화면** | Activity | `ActivityResultLauncher` 기반 이미지 로드 및 동적 UI 추가 폼 입력 구현 |

## 4. 화면 캡쳐본

### 1. 홈 화면 (Home Fragment - 전체 레이아웃 구조)
* **컴포넌트 타입:** `Fragment` (`MainActivity` 하단 탭과 연동)
* **레이아웃 구조:** * 화면 전체가 끊김 없이 스크롤되도록 최상위에 `NestedScrollView`를 배치했습니다.
  * 내부에 다양한 형태의 뷰 컴포넌트와 복수의 `RecyclerView`를 중첩(Nested)시킨 고도화된 UI 구조입니다.
* **주요 기능 및 기술 요소:**
  * **최상단 영역:** 검색창(`SearchView`) 및 자동 롤링 이벤트 배너(`ViewPager2`)
  * **중앙 퀵 메뉴:** [오늘 뭐먹지], [냉장고 털기] 등 핵심 기능 액티비티로 연결되는 5개의 이미지 버튼 레이아웃
  * **중간 콘텐츠 영역:** '내가 본 클래스', '어제 많이 본 요리' 등 가로 스크롤(`LinearLayoutManager.HORIZONTAL`)이 적용된 다수의 서브 `RecyclerView` 구현
  * **최하단 피드 영역:** 실시간 유저 요리 후기와 별점을 리스트 형태로 보여주는 세로 스크롤 `RecyclerView`

### 2. 분류 화면 (Category Fragment)
* **컴포넌트 타입:** `Fragment` (`MainActivity` 하단 탭과 연동)
* **레이아웃 구조:** * 전체 화면을 스크롤할 수 있도록 `NestedScrollView`를 최상단에 배치했습니다.
  * 대분류 타이틀(종류별, 상황별, 재료별, 방법별) 아래에 각각 독립적인 격자형 레이아웃을 구성했습니다.
* **주요 기능 및 기술 요소:**
  * **격자형 메뉴 구성:** `RecyclerView`에 `GridLayoutManager(context, 3)`를 적용하여 각 카테고리 항목을 깔끔한 3열 바둑판 배열로 시각화했습니다.
  * **데이터 필터링 연동:** 각 아이템(예: 밑반찬, 찌개 등) 클릭 시, 해당 카테고리 식별자(ID 또는 텍스트)를 다음 화면으로 전달하여 맞춤형 레시피 목록이 출력되도록 데이터 전달 아키텍처를 설계했습니다.

### 3. 혜택 화면 (Benefit Fragment)
* **컴포넌트 타입:** `Fragment` (`MainActivity` 하단 탭과 연동)
* **레이아웃 구조:** * 포인트 잔액 정보 카드, 퀵 아이콘 메뉴, 가로 배너, 상품 그리드가 복합적으로 얽혀 있는 대시보드형 화면입니다.
  * 대용량 콘텐츠를 매끄럽게 처리하기 위해 최상위를 `NestedScrollView`로 감싸 스크롤 성능을 최적화했습니다.
* **주요 기능 및 기술 요소:**
  * **동적 데이터 바인딩:** 상단 '마이 포인트' 영역에서 유저의 실시간 보유 포인트(`0P`) 등 갱신되는 텍스트 데이터를 처리합니다.
  * **하위 액티비티 라우팅:** 중간의 5개 퀵 메뉴 아이콘([출석 포인트], [쿠폰함] 등) 클릭 시 각각의 전용 서브 액티비티로 이동하는 네비게이션 로직을 포함합니다.
  * **중합 컴포넌트 배치:** 중간의 미니 게임 배너는 가로 스크롤형 뷰로 구현하고, 하단의 '포인트숍 인기쿠폰' 영역은 `GridLayoutManager(context, 2)`를 활용해 2열 상품 카드로 안정감 있게 배치했습니다.

### 4. 스크랩 화면 (Scrap Fragment)
* **컴포넌트 타입:** `Fragment` (`MainActivity` 하단 탭과 연동)
* **레이아웃 구조:** * 상단에 '전체'와 '내폴더'를 스와이프하여 전환할 수 있는 탭 구조를 가집니다.
  * 하단 영역은 사용자가 즐겨찾기(북마크)한 레시피 카드가 나열되는 리스트 구조입니다.
* **주요 기능 및 기술 요소:**
  * **상단 탭 레이아웃:** `TabLayout`과 `ViewPager2`를 연동하여 유기적인 화면 슬라이드 인터랙션을 구현했습니다.
  * **로컬 데이터 시각화 (Room DB):** 사용자가 좋아요를 누른 레시피 데이터를 로컬 DB(`Room`)에서 조회(`SELECT`)하여 `RecyclerView`에 동적으로 바인딩합니다. 
  * **커스텀 아이템 뷰:** 각 레시피 카드는 요리 썸네일(둥근 모서리 처리), 제목, 별점, 인분 및 소요 시간 텍스트 뷰를 결합한 커스텀 레이아웃으로 설계되었습니다.

### 5. MY 화면 (MY Fragment)
* **컴포넌트 타입:** `Fragment` (`MainActivity` 하단 탭과 연동)
* **레이아웃 구조:** * 최상단 유저 프로필 영역, 중간 그리드형 대시보드, 하단 메뉴 리스트가 수직으로 정렬된 구조입니다.
  * 화면 범위를 벗어나는 수많은 메뉴를 매끄럽게 보여주기 위해 `NestedScrollView`를 적용했습니다.
* **주요 기능 및 기술 요소:**
  * **하위 세로 액티비티 라우팅 (핵심 진입점):** * 초록색 **[레시피 등록]** 버튼 클릭 시 `CreateActivity`로 이동 (`ActivityResultLauncher` 연동)
    * **[내가 본 레시피]** 아이콘 클릭 시 로컬 스택에 쌓인 최근 방문 레시피 목록 화면으로 이동
    * **[설정]** 메뉴 클릭 시 앱 설정(다크모드, 알림) 화면으로 `Intent` 라우팅 처리
  * **복합 메뉴 레이아웃 수립:** 상단 통계 수치(레시피 수, 스크랩 수 등)는 `LinearLayout(horizontal)`에 가중치(`weight`)를 주어 균등 분할 배정하고, 일반 설정 항목들은 가독성이 높은 목록형 뷰로 배치했습니다.
